HTML

  • HTML is the base structure for the code and website to function.
  • The head element contains information about the webpage.
  • The body element represents the bisible content shown to the user.
  • The head and body element both contain sub-elements that help define and understand the code.

HTML sub-elements

  • "meta": Provides additional information on the HTML code such as character limitations, WIP
  • "title": Sets the name of the webpage
  • "h#": Creates a bolded and scaled font for any text put inside.
  • "img": Creates an image with attached asset.
  • "ul", "ol" ,and "li": Standing for unorderd list, ordered list, and list item, these are used in tandom to make punctiated list for the webpage.
  • "section": used to seperate parts of the webpage and well as keeping the code organized.

CSS

  • CSS is the coding for the webpage that helps with apperance.
  • CSS code can either be internal or external based on preference, however external sources must be linked with in the header of the html code.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • Git is a free open-source codeing file manager.
  • Git was made for the sole purpose of makeing a cooperative and easy way to sort files, make amendments among colleages, and keep the main code seperated from WIP code.
  • Git has multiple facets for certain purposes such as Git, the base coding language, GitHub, a place to store and amend repositories, and GitLabs, a place to test and learn about git.

Git Funcitons

  • git status: checks what directory and branch we are curently located in.
  • git checkout: allows movement between branches.
  • git chechout -b "branch-name": creates a new branch and switches to it.
